# Bitcoin CLI Reference

Throughout the rest of this course, we'll be using **bitcoin-cli** commands to interact with our own **Bitcoin regtest node**. This is a local, private blockchain that we control entirely, which means we can mine blocks on demand, broadcast transactions instantly, and inspect every detail of the chain state. It's the perfect sandbox for experimenting with Lightning transactions!

You can access your regtest node by opening the **Bitcoin Node** terminal. To do this, click **TOOLS** in the top-right corner of the screen and select **Bitcoin Node**. Once open, you can type commands directly into the terminal.

Below, you'll find the commands you'll use as you complete the exercises in this course. Don't worry about memorizing anything now, each command will be reintroduced when necessary.

### Broadcast Raw Transaction

Broadcast a raw transaction to the network.

```
sendrawtransaction <raw tx hex>
```

<details>
  <summary>What does "raw transaction" mean?</summary>

A "raw transaction" is a fully serialized Bitcoin transaction encoded as a hexadecimal string. It contains all the information needed for the Bitcoin network to validate and include the transaction in a block: the version, inputs (with signatures in the witness), outputs, and locktime. When you see a long hex string in your exercise output, that's the raw transaction ready to be broadcast!

</details>

### Decode Raw Transaction

Decode a raw transaction so that it's easy to see the various inputs, outputs, and other transaction information.

```
decoderawtransaction <raw tx hex>
```

### Get Transaction

Retrieve information about a broadcast transaction, including its confirmation count.

```
gettransaction <txid>
```

### Get UTXO

Retrieve information about a UTXO. If no information is returned, that UTXO does not exist (it has been spent or never existed).

```
gettxout <txid> <output index>
```

### Mine Blocks

Mine a specified number of blocks on the regtest network. Any transactions in the mempool will be included in the new blocks.

```
mine <number of blocks>
```

<details>
  <summary>Why mine blocks manually?</summary>

On Bitcoin's mainnet, blocks are mined roughly every 10 minutes by miners competing to solve a proof-of-work puzzle. On regtest, there are no miners, so blocks are only created when you explicitly tell the node to mine them. This gives us precise control over the blockchain, which is perfect for testing. We can broadcast a transaction and then mine exactly the number of blocks we need to confirm it.

</details>

### Getting Help

If you ever forget a command or want to explore what else is available, you can type `help` in the Bitcoin Node terminal to see all available commands.

```
help
```
