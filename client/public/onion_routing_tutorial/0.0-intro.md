# Onion Routing and Lightning Payments

Welcome to **Onion Routing and Lightning Payments**, an interactive course on how the Lightning Network routes payments privately across a network of nodes.

In this course, we'll build the cryptographic machinery that powers Lightning's payment routing from the ground up. You'll learn how a sender can construct an encrypted **onion packet** that traverses multiple hops, where each hop can only see the minimum information needed to forward the payment, and nothing more.

## What You'll Learn

Over the next sections, we'll cover:

- **What each hop knows** and, crucially, what it *doesn't* know
- How to **find a route** through the Lightning Network and calculate fees and timelocks
- The **cryptographic primitives** behind onion routing: shared secrets, key derivation, and session key blinding
- How to **build an onion packet** layer by layer using the Sphinx protocol
- How to **peel an onion layer** from the perspective of a forwarding node
- The **message choreography** that keeps channels in sync during payment forwarding
- How **success and failure** propagate back through the route
- A hands-on **Payment Trace Lab** where you'll trace a complete payment end to end

## How This Course Works

This course is fully interactive. You'll complete **coding exercises** in Python (running directly in your browser), answer **knowledge checks** along the way, and earn **sats** for correct solutions.

Each section also features **interactive diagrams** that you can explore from different perspectives. A key theme of this course is *privacy*: what information is visible to each node in the network? You'll be able to toggle between an **omniscient view** (seeing everything) and a **node-local view** (seeing only what a specific node knows).
