# Capstone: Successful Payment

You've made it! Over the last eleven chapters, you wrote every function a Sphinx onion needs: the shared-secret chain, the key derivation, the filler, the wrap, the peel, the HMAC verification, and the forwarding checks. Now it's time to see everything come to life! In this capstone, you'll watch an onion packet get built and routed across the entire payment path from Alice to Dave using your code (yes, the exact code you wrote in the exercises!). And the best part, you'll see everything debugger-style, so you can step through each and every line of code (if you want!).

## What happens when you click "Run the capstone"

1. **Preflight.** First, the lab will re-run all nine exercise test suites against your code, ensuring that you've successfully completed each exercise. If something fails, the panel lists exactly which exercise you need to revisit.
2. **Assembly.** Once your functions are passing, your builder methods (`derive_shared_secrets`, `generate_filler`, `wrap_hop`, `build`) will be stitched into `sphinx/builder.py`, and your forwarder functions (`verify_hmac`, `peel_layer`, `check_forward`) into `sphinx/forwarder.py`. These are the two "files" you'll see in the lab code pane.
3. **The run.** Finally, a fixed route (Alice → Bob → Charlie → Dave, paying Dave 1,000,000 sats) will run entirely in your browser using *your* code.

The lab records every step of that run. Once it finishes, you're in the driver's seat... step forward, step back, jump around, and watch the onion change exactly when your code changes it.
